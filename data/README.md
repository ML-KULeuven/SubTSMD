## Synthetic

The synthetic data consists of Gaussian noise, in which simple shapes are injected, such as the period of a sine-wave, an increasing trend, ... This univariate patterns are combined to cover multiple attributes, creating subspace motifs. The different sets vary in the dimensionality of the data, and the length of the individual motifs. 

Synthetic data can be generated as follows:
```python 
import numpy as np
from sub_tsmd import generate

X, y = generate(
    dimension=6,
    nb_motif_sets=5,
    min_motif_set_size=3,
    max_motif_set_size=5,
    min_motif_dimension=2,
    max_motif_dimension=3,
    univariate_motifs=[
        lambda x: np.sin(np.linspace(0, 2 * np.pi, x)),  # Sinusoidal 
        lambda x: np.concatenate([np.ones(int(np.ceil(x/2))), -1 * np.ones(int(np.floor(x/2)))]),  # Square wave
        lambda x: np.exp(-np.linspace(0, 2, x)) * np.sin(2 * np.pi * np.linspace(0, 1, x)),  # Damped sine
        lambda x: np.cos(np.linspace(0, 2 * np.pi, x)),  # Cosine wave
        lambda x: np.linspace(-1, 1, x),  # Linear increasings
        lambda x: np.sin(4 * np.pi * np.linspace(0, 1, x)),  # Higher frequency sine
        lambda x: np.sign(np.sin(np.linspace(0, 4 * np.pi, x))),  # Square wave-like signal
        lambda x: np.sin(4 * np.pi * np.linspace(0, 1, x)),  # Higher frequency sine
        lambda x: np.linspace(1, -1, x)  # Linear decreasing
    ], 
    min_motif_length=45,
    max_motif_length=55,
    white_space=0, 
    nb_motif_repositions=1,
    noise_general=0.05,
    noise_non_motif=1.5,
    seed=0
)
```

## Motif pairs

The motif pairs is generated similarly as the synthetic data, but in which we force that there is only a single motif set (``nb_motif_sets=1``) which consists of exactly two motifs (``min_motif_set_size=2`` and ``max_motif_set_size=2``). The remaining variables are changed accordingly. 

## Sub-tsmd-bench

This dataset is generated by concatenating the data from different classes from time series classification benchmarks. The instances of a single class correspond to the motifs of a motif set. The generation-code is based on [1], which ensures that the number of _off-target_ patterns is kept to a minimum. Within our implementation, we increased the dimension of the time series to allow the classification instances to span only a subspace of the data. The unused dimensions are filled by spurious patterns, similarly to those that are interleaved between the different motifs. While this benchmark set consists of only real data, two motifs spanning a different subspace can not overlap. 

## Noisy-attributes

This set takes as input the synthetic data, and adds a number of random walk attributes to the data [3, 4]. Due to their randomness, these attributes should not contain any meaningful motifs, and are therefore irrelevant. 

## Noisy-data

This set takes as input the synthetic data, and adds Gaussian noise on top of the data, in which the standard deviation of the noise is set to a percentage of the standard deviation of the original data [5].

## Scalability

This set is generated for measuring the scalability of the models. The time series vary in length and number of attributes. There is also ground truth available, but this is less relevant. The data is generated using the synthetic data generator described above. However, the generator can not guarantee that the length of the time series has a specific value. This has been enforced in this series, to facilitate easy interpretation and more direct comparison of the runtimes across different time series and models. 

## Physical-therapy

The dataset contains wearable sensor data recording patients doing 8 physical exercises [2]. For each exercise, 5 sensors were used, which each have a accelerometer, gyroscope and magnetometer measuring in X, Y and Z direction. For each exercise, only a subset of the sensors are relevant (e.g., the sensors on the left leg are not relevant when doing an exercise with the right leg). 

The data can be found on the following page: https://doi.org/10.13140/RG.2.2.20101.01768. You should download ``physical_theraby_dataset_Yurtman_Barshan.zip`` and ``physical therapy - only labels.zip``, extract their content, and organize the files as follows: 
```
physical-therapy/raw-data
в”њв”Ђв”Ђ s1
в”њв”Ђв”Ђ s2
в”њв”Ђв”Ђ s3
в”њв”Ђв”Ђ s4
в”њв”Ђв”Ђ s5
в””в”Ђв”Ђ labels
```
The folders ``s1`` until ``s5`` come from ``physical_theraby_dataset_Yurtman_Barshan.zip``, and contain the sensor data of each subject. We used subject 1 (``s1``) for validation and tuning parameters, the remaining subjects were used for testing. The ``labels`` folder comes from ``physical therapy - only labels.zip`` and contains the annotation files. 

The ``physical-therapy/generate.ipynb`` notebook will read this data, and format it such that it can be loaded as indicated above.

## PAMAP2

The PAMAP2 dataset [6] consists of wearable sensor data, and records 9 subject performing a series of 18 activities using 3 IMUs and 1 heart rate sensor. The data is publicly available [here](https://archive.ics.uci.edu/dataset/231/pamap2+physical+activity+monitoring). Download the data, unzip the files, and make sure you have the following structure:
